# Relational Documents - PDF文書マークアップシステム

膨大なPDF文書にマークづけをしていくことで、それらの関連性を閲覧できるシステム。関連性をつけることで文書内の様々な情報の関係や更新による変化、事前に設定した条件との整合性を瞬時に把握できます。

## 機能

### 📄 文書の閲覧・管理

- PDF文書の登録・閲覧
- 複数の表示モード（リッチ表示、リスト表示）
- 文書のジャンル・タグ管理
- 改訂履歴の管理

### 🔍 マークアップ機能

- PDF上の任意の箇所にマークアップを追加
- マーカーのスタイル（ハイライト、直線、ボックス、アンダーライン）
- マークアップ同士のリンク付け
- マークアップの内容・タグ管理

### 📊 改訂履歴・整合性管理

- 文書ごとの改訂履歴管理
- マークアップブロック単位での変更追跡
- 整合性チェックルール設定
- 関連文書間の連携

### ⚙️ システム設定

- ダークモード対応
- 表示モード切り替え（リッチ/リスト）
- ソート設定（名前/更新日時/ジャンル）
- 多言語対応（日本語/英語）

## システム構成

### アーキテクチャ

```
フロントエンド (Vue3 + TypeScript)
     ↓
API層 (関数呼び出し統合型)
     ↓
サービス層 (ビジネスロジック)
     ↓
リポジトリ層 (データ永続化)
     ↓
IndexedDB (ブラウザ内ストレージ)
```

### ディレクトリ構造

```
src/
├── apis/                    # API層（フロントエンド統合）
│   └── backendApi.ts
├── services/                # ビジネスロジック層
│   └── documentService.ts
├── repositorys/            # データ永続化層
│   └── localStorageRepository.ts
├── models/                  # Zodスキーマ定義
│   └── schemas.ts
├── pages/                   # ページコンポーネント
│   ├── IndexPage.vue       # 文書一覧
│   ├── DocumentViewerPage.vue  # 文書表示
│   └── SettingsPage.vue    # 設定
├── layouts/                 # レイアウトコンポーネント
│   └── MainLayout.vue
├── components/              # UI コンポーネント
├── i18n/                    # 多言語対応
│   ├── en-US/
│   └── ja-JP/
├── utils/                   # ユーティリティ
│   ├── appInitializer.ts   # アプリ初期化
│   └── systemTest.ts       # システムテスト
└── boot/                    # Boot スクリプト
    ├── i18n.ts
    └── appInit.ts
```

## インストール・実行

### 前提条件

- Node.js 22以上
- Bun 1.3以上

### 依存パッケージ

```bash
bun install
```

### 開発サーバーの起動

```bash
bun run dev
```

ブラウザが自動で開き、アプリケーションが起動します。

### ビルド

```bash
bun run build
```

生成されたファイルは `dist/` ディレクトリに出力されます。

### コード品質チェック

```bash
# Linting
bun run lint

# Format
bun run format
```

## 使用方法

### 1. ドキュメント一覧ページ

- アプリ起動時に表示されるページ
- ドキュメントの追加・編集・削除
- 表示モード・ソート設定
- キーワード検索

### 2. ドキュメント表示ページ

- クリックしたドキュメントの詳細表示
- **左パネル**: PDF表示エリア（ズーム機能付き）
- **右パネル**: マークアップ・改訂履歴・情報表示

**マークアップ操作**:

- 「新規マークアップ」ボタンでマークアップを追加
- マークアップをクリックして選択・編集
- 編集内容を「保存」ボタンで確定
- 「削除」ボタンでマークアップを削除

### 3. 改訂履歴タブ

- 文書の変更履歴を一覧表示
- 改訂番号・変更日時・説明を表示

### 4. 文書情報タブ

- ドキュメント名
- ジャンル・ページ数
- 作成日時・更新日時
- 説明文

### 5. 設定ページ

- ダークモード切り替え
- 表示モード選択
- ソート順序設定
- 言語選択
- テストデータ生成（デモ用）

## データ永続化

システムはブラウザ内のIndexedDBにすべてのデータを保存します：

- **documents**: 文書メタデータ
- **markups**: マークアップ情報
- **revisions**: 改訂履歴
- **markupBlocks**: マークアップブロック
- **complianceRules**: 整合性ルール
- **folders**: フォルダ構造
- **settings**: アプリケーション設定

## 開発ガイドライン

### コード スタイル

- ファイル名: `camelCase.ts` 形式
- コメント言語: 日本語
- Docstring形式の関数説明
- TypeScript strict mode

### 型管理

- Zodでスキーマ定義
- すべての型定義は `src/models/schemas.ts` で一元管理

### API設計

- フロントエンド ← → API層（関数呼び出し）
- API層 ← → サービス層（ビジネスロジック）
- サービス層 ← → リポジトリ層（データ操作）

### 多言語対応

- すべての表示テキストはi18nを通じて表示
- `src/i18n/` で言語ファイルを管理

## トラブルシューティング

### IndexedDB が初期化できない

- ブラウザのキャッシュをクリアしてリロード
- プライベートウィンドウ（シークレットモード）では動作しません

### マークアップが表示されない

- ドキュメントが正しく読み込まれているか確認
- ページ番号が正しく設定されているか確認

### 設定が保存されない

- ブラウザのストレージ容量確認
- ブラウザ設定で IndexedDB が有効になっているか確認
